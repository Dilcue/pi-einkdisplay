# Web UI — Design Spec

**Date:** 2026-03-28
**Status:** Approved

---

## Overview

A Flask-based settings UI running on the Pi at `http://papirus.local:8080`. Dashboard layout with four status cards — Calendar, Weather, Display, System. Changes save to `config.json` / `.env` and automatically restart the `einkdisplay` systemd service.

---

## Architecture

Two systemd services on the Pi:

- **`einkdisplay.service`** — existing display loop (`main.py`)
- **`einkdisplay-web.service`** — new Flask app (`web/app.py`, port 8080)

Both read from the same `config.json` and `.env`. The web service writes config changes then calls `systemctl restart einkdisplay`. A crash in the web UI never affects the display.

No authentication — LAN-only access.

---

## Config Changes

`calendar_ids` moves from hardcoded in `data/calendar_client.py` into `config.json` so the web UI can manage it:

```json
{
  "location_name": "Your City, ST",
  "latitude": "YOUR_LATITUDE",
  "longitude": "YOUR_LONGITUDE",
  "calendar_display_name": "Family Calendar",
  "calendar_ids": [
    "your-email@example.com",
    "your-family@example.com",
    "your-calendar-id@group.calendar.google.com"
  ],
  "calendar_max_events": 3,
  "page_delay_seconds": 7,
  "data_refresh_minutes": 60,
  "pages": ["clock", "weather_current", "weather_forecast", "calendar"]
}
```

`data/calendar_client.py` reads `settings.calendar_ids` instead of the hardcoded list.

## Sections

### Calendar (`/calendar`)

- Checkbox list of all calendars on the account (fetched live from Google Calendar API)
- Number of events to show (1–5, dropdown)
- OAuth status: connected account email + "Re-authorize" button
  - Re-auth triggers a standard OAuth web flow: browser redirects to Google, callback handled by Flask at `/oauth/callback`, token saved to `token.json`
- Save writes `calendar_ids` and `calendar_max_events` to `config.json`, restarts display

### Weather (`/weather`)

- Location name (display label only)
- Latitude and longitude
- OWM API key (shown masked, editable)
- Data refresh interval (dropdown: 15 / 30 / 60 / 120 min)
- Save writes to `config.json` and `.env`, restarts display

### Display (`/display`)

- Page delay in seconds (numeric input)
- Active pages with enable/disable checkboxes and drag-to-reorder
- Page order persisted to `config.json` as `pages` array
- Save writes `config.json`, restarts display

### System (`/system`)

- Service status (running / stopped)
- Uptime
- Last weather fetch time
- Last calendar fetch time
- "Restart Display" button — calls `systemctl restart einkdisplay`
- "View Logs" — shows last 50 lines from `journalctl -u einkdisplay`

---

## Service Restart

All save actions:
1. Write updated values to `config.json` and/or `.env`
2. Call `subprocess.run(['sudo', 'systemctl', 'restart', 'einkdisplay'])`
3. Redirect back with a success flash message

The web service user needs a passwordless sudoers entry for the restart command.

---

## systemd Service

`/etc/systemd/system/einkdisplay-web.service`:

```ini
[Unit]
Description=E-Ink Display Web UI
After=network.target

[Service]
User=chris
WorkingDirectory=/home/chris/einkdisplay
ExecStart=/home/chris/einkdisplay-venv/bin/python web/app.py
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

## Out of Scope

- Password / authentication
- HTTPS
- Mobile-specific layout (readable on phone but not optimized)
- Live display preview
- Weather location search (coordinates entered manually)
